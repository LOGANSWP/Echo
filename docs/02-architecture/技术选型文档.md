# Echo · 回响：端侧 AI 技术选型决策记录

**版本**：v4.6

**更新日期**：2026-06-10

**适用范围**：iOS 18+ 端侧多模态检索、语音转写、文本嵌入、向量存储、推理框架及应用架构

**决策原则**：隐私优先 > 性能/功耗平衡 > 生态兼容性 > 模型精度上限

**对齐规格**：Echo v4.6 全量用户故事与验收标准规格书

## 1. 视觉编码模型 (Vision Encoder)

用于图片关键帧与视频截帧的语义向量化，是跨模态检索的视觉基石。v4.6 要求模型随 App 分发，无网络下载；支持低电量/过热降级切换。

| 选项                       | 优势                                                         | 劣势                         | 适用场景                    | 决策状态     |
| -------------------------- | ------------------------------------------------------------ | ---------------------------- | --------------------------- | ------------ |
| **SigLIP-SO400M (INT4)**   | 图文对齐效果 SOTA；支持动态分辨率；量化后 ~250MB；推理 <120ms (A18 Pro) | 需预先转换为 Core ML 格式    | 主力检索模型（随 App 内置） | ✅ **首选**   |
| **MobileCLIP-B**           | Apple 优化；体积小 (~300MB)；推理快                          | 中文召回率低于 SigLIP        | 低电量/过热降级模型         | 🔄 备选(降级) |
| **Apple Vision Embedding** | 系统内置零体积；隐私合规                                     | 向量空间不与文本对齐；黑盒   | 不适用跨模态检索            | ❌ 不推荐     |
| **Qwen2-VL-2B-Instruct**   | 生成式理解强；中文顶尖                                       | 内存占用 >1.5GB；延迟 >2s/帧 | 按需深度分析（非实时检索）  | 🔬 研究储备   |

### 💡 决策依据 (v4.6 更新)

- **SigLIP-SO400M INT4 量化版** 作为唯一主力视觉模型，**随 App 安装包分发**，不依赖任何网络下载。模型文件置于 `Bundle.main` 资源目录，使用 Core ML 加载。首次启动即可用，无 ODR 或额外下载流程。
- **降级策略**：根据 `ProcessInfo` 低电量模式或热状态 (`thermalState`) 自动切换至 MobileCLIP-B。切换逻辑在 `VisionPipeline` Actor 中实现，并触发 UI 警告 Banner（符合 US-RES-002/003）。
- **模型加载失败处理**：若模型文件损坏或权限问题，显示“功能受限”提示，并提供“重试加载”按钮（仅重新加载本地文件，不发起网络请求）。无定时自动重试（符合 US-RES-004）。

## 2. 语音转写模型 (ASR)

用于语音备忘录离线转写。v4.6 要求转写仅本地完成，原始音频不持久化。

| 选项                         | 优势                                         | 劣势                 | 适用场景                    | 决策状态         |
| ---------------------------- | -------------------------------------------- | -------------------- | --------------------------- | ---------------- |
| **Whisper.cpp (base INT4)**  | 开源可控；量化后 ~150MB；支持语种检测/时间戳 | 中文标点需后处理     | 主力转写引擎（随 App 内置） | ✅ **首选**       |
| **Apple SFSpeechRecognizer** | 系统级优化；零体积；实时流式                 | 离线精度低于 Whisper | 实时预览辅助                | 🔄 备选(实时预览) |
| **Whisper.cpp (tiny)**       | 体积极小 (~75MB)                             | 中文错字率高 >15%    | 不满足精度要求              | ❌ 不适用         |

### 💡 决策依据 (v4.6 更新)

- **Whisper.cpp base INT4** 随 App 内置，首次启动即可用。1 分钟音频转写 <6s (A18 Pro)，中文 CER <8%。
- **双引擎策略**：录音时使用 `SFSpeechRecognizer` 提供实时预览文字；保存后异步调用 Whisper.cpp 精转，融合结果作为最终记忆文本。原始音频不存储，仅保留转写文本（符合 US-ING-003）。
- **转写后处理**：自研标点恢复、数字规范化 Pipeline，提升检索友好度。

## 3. 文本嵌入模型 (Text Embedding)

用于备忘录、转写文本、查询语句的语义向量化。v4.6 强调中英双语对齐，支持指令感知检索。

| 选项                               | 优势                                          | 劣势                          | 适用场景                        | 决策状态     |
| ---------------------------------- | --------------------------------------------- | ----------------------------- | ------------------------------- | ------------ |
| **GTE-Qwen2-1.5B-Instruct (INT4)** | 中英 SOTA；指令感知；与 SigLIP 共享 CLIP 空间 | 量化后 ~900MB；单次推理 ~50ms | 唯一文本嵌入模型（随 App 内置） | ✅ **首选**   |
| **Apple Embedding Model**          | 零体积；系统内置                              | 中文语义粗；不支持指令        | 仅用于元数据预过滤              | 🔄 备选(辅助) |
| **bge-m3-zh**                      | 中文专精；多表示                              | 无法与视觉模型对齐            | 破坏架构一致性                  | ❌ 不适用     |

### 💡 决策依据 (v4.6 更新)

- **GTE-Qwen2-1.5B-Instruct INT4** 随 App 内置，无网络下载。与 SigLIP 共享向量空间，实现“中文输入、英文记忆”互检（符合 US-RET-001/002）。
- **指令感知**：支持情感查询（如“找悲伤时刻”），配合 US-SYN-006 情绪干预模式。
- **降级处理**：模型加载失败时，仅提供 FTS5 关键词检索（符合 US-RES-004）。

## 4. 端侧向量数据库 (Vector Store)

存储与检索多模态向量。v4.6 要求支持增量更新、加密、断点续传进度存储（SQLite）。

| 选项                    | 优势                                                    | 劣势                      | 适用场景   | 决策状态   |
| ----------------------- | ------------------------------------------------------- | ------------------------- | ---------- | ---------- |
| **LanceDB Mobile**      | 嵌入式零依赖；增量无锁；列式压缩；支持 NSFileProtection | iOS 生态较新              | 主向量存储 | ✅ **首选** |
| **Chroma Lite**         | Python 生态易迁移                                       | iOS 编译偶发问题          | 备选       | 🔄 备选     |
| **SQLite + sqlite-vss** | 事务安全                                                | 扩展维护停滞；不支持增量  | 风险高     | ❌ 不推荐   |
| **Faiss Mobile**        | 检索速度极致                                            | 无持久化/加密；不支持增量 | 不适用     | ❌ 不推荐   |

### 💡 决策依据 (v4.6 更新)

- **LanceDB Mobile** 作为唯一向量库，随 App 沙盒创建。支持 `NSFileProtectionComplete`，符合隐私要求。
- **增量写入**：无需重建全量索引，满足用户故事 US-SRC-012（数据源变更自动同步）的实时性。
- **断点续传辅助**：长任务（如全量索引构建）的进度使用独立 **SQLite 表 `TaskProgress`** 存储，而非 LanceDB。进度记录在任务完成后自动删除，符合 US-SYS-001。

## 5. 推理执行框架 (Inference Runtime)

统一调度模型加载、量化、缓存与硬件加速。v4.6 明确移除 ODR，所有模型随 App 分发。

| 选项                       | 优势                                 | 劣势                           | 适用场景                        | 决策状态           |
| -------------------------- | ------------------------------------ | ------------------------------ | ------------------------------- | ------------------ |
| **Core ML**                | Apple 原生优化；ANE/GPU/CPU 自动调度 | 转换工具链偶发 bug             | 所有生产模型部署（视觉/嵌入）   | ✅ **主力运行时**   |
| **MLX**                    | Apple 开源；Unified Memory 利用率高  | 与 Core ML 共存时内存管理复杂  | 未来统一推理层（评估中）        | 🔄 准生产备选       |
| **llama.cpp**              | LLM/ASR 专精优化；量化丰富           | 仅适用特定模型                 | ASR 专用                        | ✅ **ASR 专用补充** |
| **Apple Intelligence SDK** | 系统级集成                           | 黑盒不可审计；不可用于认知管线 | 仅限 Writing Tools 等非敏感场景 | ⚠️ **受限集成**     |

### 💡 决策依据 (v4.6 更新)

- **所有模型（SigLIP、MobileCLIP、GTE-Qwen2、Whisper）均随 App 安装包分发**，存放于 `Bundle.main`。App 内不发起任何网络下载请求，彻底移除 ODR（符合 US-RES-004）。
- **模型加载失败**：不提供自动重试，仅在 UI 上显示“重试加载”按钮，重新尝试加载本地文件。用户也可跳转系统设置修复 App（符合 US-RES-004）。
- **MLX 评估**：若 iOS 19 上 MLX 稳定性进一步提升且内存管理优于 Core ML，可考虑 V5.0 迁移。

## 6. 应用架构与状态管理 (App Architecture)

v4.6 延续 `@Observable + Actor` 模式，增强隐私校验、并发安全与统一错误处理。

| 选项                                 | 优势                                                | 劣势                            | 适用场景         | 决策状态   |
| ------------------------------------ | --------------------------------------------------- | ------------------------------- | ---------------- | ---------- |
| **@Observable + Actor 隔离**         | 原生高性能；精确刷新；与 Swift Concurrency 无缝集成 | 需自建副作用审计                | 所有业务管线     | ✅ **首选** |
| **TCA**                              | 单向数据流；Effect 可测试                           | 样板代码多；与 @Observable 重叠 | 不推荐用于新代码 | ❌ 过渡     |
| **SwiftUI + @Observable (无 Actor)** | 最简写法                                            | 并发不安全；副作用不可审计      | 纯展示组件       | ⚠️ 受限     |
| **Combine + MVVM**                   | 旧教程多                                            | 测试困难；内存泄漏              | 不用于新开发     | ❌ 已淘汰   |

### 💡 决策依据 (v4.6 更新)

- **统一错误响应矩阵**：工程中实现 `ErrorHandler` 协议，所有异步操作按 L1~L4 等级分类处理。L2 错误写入 `PendingOperations` 表，仅支持用户手动重试（无自动重放）。L1 重试 3 次后升级为 L2。
- **隐私校验 `PrivacyCheckpoint`**：每个 Pipeline 方法（如 `search`, `ingest`）必须调用 `PrivacyCheckpoint.validate()`，CI 强制覆盖率 100%。
- **断点续传**：使用 SQLite `TaskProgress` 表存储进度；任务完成后自动删除记录。取消任务时保存断点，下次启动时询问是否继续（符合 US-SYS-001）。
- **导航**：`NavigationStack` + `NavigationPath`，支持状态恢复。

## 7. 国际化与动态内容本地化 (i18n & Dynamic Localization)

v4.6 强调仅支持简体中文和英文，AI 输出语言与 UI 统一。

| 层级            | 技术方案                            | 适用范围             | 决策状态   |
| --------------- | ----------------------------------- | -------------------- | ---------- |
| **静态 UI**     | Xcode String Catalogs (.xcstrings)  | 按钮、标题、提示     | ✅ 标准     |
| **AI 动态内容** | Prompt 约束 + 结构化输出 + 语种校验 | 搜索结果、卡片、摘要 | ✅ 核心     |
| **数据存储**    | GTE-Qwen2 双语嵌入 + 源语言元数据   | 跨语言检索           | ✅ 架构内建 |
| **运行时翻译**  | 禁用任何第三方/端侧翻译服务         | 无                   | ❌ 明确排除 |

### 💡 决策依据 (v4.6 更新)

- `preferredLanguage` 仅允许 `zh-Hans` 或 `en-US`。系统语言为繁体中文或其他方言时，默认 `zh-Hans`，首次启动提示一次。
- **AI 输出三重保障**：Prompt 注入语言参数 → 结构化 JSON 携带 `responseLanguage` → 后处理使用 `NaturalLanguage` 语种识别，不匹配则重试一次或返回模板回复。
- **跨语言检索**：依赖 GTE-Qwen2 的双语对齐能力，无需翻译层。

## 8. 本地反馈学习机制 (Feedback Learning)

v4.6 新增纯本地反馈学习，支持点赞/点踩、时间衰减、权重截断。

| 组件          | 技术选型                                                     | 说明                  |
| ------------- | ------------------------------------------------------------ | --------------------- |
| **反馈存储**  | SQLite (FeedbackStore)                                       | 纯本地，永不上传      |
| **重排公式**  | `finalScore = cosineSim + clamp(Σ(👍*decay) - Σ(👎*decay), -0.5, 0.5)` | 阈值 ≥0.80 才应用反馈 |
| **时间衰减**  | 年龄 >90天 ×0.5；>180天归档                                  | 每条反馈独立计算      |
| **重置/撤销** | 清除表或删除单条记录                                         | 即时生效              |

### 💡 决策依据

- 完全符合 US-FBK-001/002/003，所有反馈数据仅存本地 SQLite，不设计任何上传逻辑。
- 余弦相似度阈值 0.80 及权重上限 ±0.5 在 `Retriever` 中实现，确保反馈学习稳定。

## 9. 后台任务与断点续传 (Background Tasks)

v4.6 明确要求后台任务面板、串行队列、SQLite 进度存储。

| 需求             | 技术方案                                                  |
| ---------------- | --------------------------------------------------------- |
| **后台任务注册** | `BGAppRefreshTask`, `BGProcessingTask`                    |
| **任务串行队列** | 自定义 `AsyncStream` + `Actor` 确保索引构建与数据同步互斥 |
| **进度存储**     | SQLite 表 `TaskProgress`（独立于向量库）                  |
| **断点续传**     | 取消时保存 `lastProcessedIndex`；重新开始询问用户         |
| **清理策略**     | 任务完成后自动删除进度记录                                |

### 💡 决策依据

- 符合 US-SYS-001。暂停/取消操作通过 `Task` 取消句柄实现。
- 进度展示使用 `Progress` 类型，绑定到 UI 状态。

## 10. 地理围栏与情境唤醒 (Geofencing)

v4.6 简化围栏重置规则：仅基于系统 `didExit` 后 `didEnter` 重置，永久停留不再重复推送。

| 组件         | 技术方案                                                     |
| ------------ | ------------------------------------------------------------ |
| **围栏监控** | `CLCircularRegion` + `CLLocationManager`                     |
| **重置逻辑** | 记录上次推送时间；收到 `didExit` 清除标记；`didEnter` 且无最近推送时触发 |
| **防打扰**   | 同一围栏 24h 内不重复（若未离开）                            |
| **权限关闭** | 静默禁用                                                     |

### 💡 决策依据

- 符合 US-AWK-001。不需要 30 秒持续离开判断，依赖 iOS 系统区域事件防抖。
- 推送内容匹配度 ≥0.7 由检索引擎计算。

## 11. 音乐推荐 (Music Suggestion)

v4.6 采用内置离线热门歌曲库，不依赖网络。

| 选项                | 优势                      | 劣势                      | 决策状态               |
| ------------------- | ------------------------- | ------------------------- | ---------------------- |
| **Apple Music API** | 个性化推荐质量高          | 需用户授权+订阅；API 限流 | 可选（若授权）         |
| **内置离线歌单**    | 零网络；稳定；随 App 更新 | 内容固化，无实时性        | ✅ **备选方案（主力）** |

### 💡 决策依据

- 内置离线热门歌曲库：按年份分类，每年 20 首，随 App 版本更新（JSON 文件）。不依赖网络，符合本地优先原则。
- 若用户授权 Apple Music，优先使用 API 推荐，API 限流时静默降级至离线库。

## 14. 附录：基准测试环境与方法论（v4.6 更新）

### 测试设备矩阵

- **旗舰基准**：iPhone 16 Pro (A18 Pro, 8GB RAM)
- **主流基准**：iPhone 15 (A16, 6GB RAM)
- **入门基准**：iPhone 16e (A18, 8GB RAM)

### 评测指标定义

- **检索质量**：Recall@10 / NDCG@10（人工标注 500 条跨模态查询）
- **推理延迟**：P50/P95/P99（预热后连续 100 次）
- **内存峰值**：Instruments Allocations（含模型加载+推理缓冲）
- **功耗影响**：Xcode Energy Gauge（后台批量处理）
- **存储效率**：每千条记录向量库文件大小 + SQLite 进度表开销
- **语言一致性**：AI 输出语言与 `preferredLanguage` 匹配率 ≥99%
- **隐私校验覆盖率**：100%
- **反馈重排准确率**：人工评估反馈后检索结果改善比例

### Agent 产出质量门禁
